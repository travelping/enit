Vision

Lightweight, dynamic alternative to OTP releases, with some key differences

  • Uses shared Erlang VM installed through standard mechanism
  • Applications can be upgraded individually, no release update required
  • Bootstrap release configuration, reload changed config from disk
  • Single CLI tool with everything needed to run a release

Architecture

The CLI tool is implemented as an Escript.

Starting a release

  • exec new VM
  • load OTP apps
  • set environment parameters
  • launch apps in dependency order

Stopping a release

  • init:stop/1

Reconfigure

  • diff running config and config on disk
  • set changed parameters
  • call config_change/3 in applications whose parameters have changed
